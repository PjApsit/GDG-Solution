# NGO Decision Intelligence System - Setup Guide

Welcome to the project! Follow these steps to get the project running locally.

## Prerequisites
- Node.js version 20 installed on your system.
- Git installed on your system.

## Setup Steps

1. **Clone the Repository**
   Open your terminal and clone the repository to your local machine:
   git clone <your-repository-url>
   cd "GDG Solution"

2. **Install Client Dependencies**
   Navigate to the client folder and install required packages:
   cd client
   npm install
   cd ..

3. **Install Server Dependencies**
   Navigate to the server folder and install required packages:
   cd server
   npm install
   cd ..

4. **Environment Variables**
   Copy the example env files and fill in the real values:
   ```bash
   copy client\.env.example client\.env
   copy server\.env.example server\.env
   ```

   Client `client\.env` should include Firebase web config values:
   - `VITE_FIREBASE_API_KEY`
   - `VITE_FIREBASE_AUTH_DOMAIN`
   - `VITE_FIREBASE_PROJECT_ID`
   - `VITE_FIREBASE_STORAGE_BUCKET`
   - `VITE_FIREBASE_MESSAGING_SENDER_ID`
   - `VITE_FIREBASE_APP_ID`
   - `VITE_GEMINI_API_KEY` (optional for direct client-side calls)

   Server `server\.env` should include:
   - `PORT=3001`
   - `GEMINI_API_KEY`
   - `FIREBASE_PROJECT_ID`
   - `FIREBASE_CLIENT_EMAIL`
   - `FIREBASE_PRIVATE_KEY`

   If the Firebase server variables are missing, the backend falls back to dummy in-memory data.

## Running the Application

You will need to run both the frontend and backend simultaneously. Open TWO separate terminal windows.

**Terminal 1 (Backend API)**
1. Navigate to the server folder:
   cd server
2. Start the development server:
   npm run dev
(The server should now be running on http://localhost:3001)

**Terminal 2 (Frontend React App)**
1. Navigate to the client folder:
   cd client
2. Start the development server:
   npm run dev
(The frontend should now be running on http://localhost:5173)

**Finally:**
Open http://localhost:5173 in your browser to view the application!

## Available APIs
- `GET /api/health`
- `GET /api/events`
- `GET /api/tasks`
- `POST /api/scan/survey`
- `POST /api/scan/batch`
- `GET /api/matching/task/:taskId`
- `GET /api/matching/volunteer/:volunteerId`
- `GET /api/export/events`
- `GET /api/export/tasks`
- `POST /api/predict/needs`
- `GET /api/volunteers`
- `GET /api/volunteers/:id`
- `GET /api/volunteers/suggested/:taskId`

## Notes
- The app currently uses mock data in several UI pages, so some screens are demo-backed rather than fully connected to Firestore.
- There is no backend social route mounted yet, even though the client includes social UI and helper functions.
